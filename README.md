# LingoFlow 🎙️

自分専用のAI英語スピーキング練習アプリ。完全無料構成（Gemini API無料枠 + Render無料プラン、クレジットカード登録不要）。

## 機能（v1.3）

- **シナリオ練習**: 5カテゴリ（基礎・旅行・日常生活・仕事・トラブル対応）**全15シナリオ**を
  **①シャドーイング → ②ロールプレイ会話 → ③AIフィードバック** の3ステップで練習
- **レベル選択**: シナリオ開始前に**初級（A2）／中級（B1-B2）／上級（B2-C1）**を選択。
  フレーズの難しさ・AIの話し方・フィードバックの着眼点・お手本音声の速さが変わる
- **AIフリートーク**: 好きな話題で自由会話。**音声モード／チャットモード**切替対応（通勤中はチャットで）。
  AIの返答は**「訳を見る」トグル**で日本語訳を確認できる（デフォルトは英語のみで没入感を維持、
  フリートーク・シナリオ内ロールプレイの両方に対応）
- **シナリオ内容のキャッシュと生成済み表示**: シナリオ×レベルの組み合わせごとに内容をキャッシュし、
  無駄なAI呼び出し（トークン消費）を削減。**生成済みのレベルにはゴールドのドット**が付き、
  追加消費なしですぐ始められることが分かる。「新しい内容で作り直す」で明示的に再生成も可能
- **表現集**: あいづち・聞き返し・意見・仕事・トラブル対応など**9カテゴリ約80表現**を頻度（ゴールドドット3段階）付きで収録。
  タップで発音を確認できる。さらに**フィードバックで指摘された言い回しが「あなたの表現」に自動で蓄積**され、
  同じ指摘が繰り返されると頻度カウント（×n）が上がる
- **過去セッションの全文閲覧**: 記録画面から過去の会話（オープントーク・シナリオ内ロールプレイ両方）をタップして、
  全文（読み返し専用）とその時のフィードバックを確認できる
- **発話の送信前確認**: 音声入力は認識終了後すぐに送信せず、**文字起こしプレビュー → 自分の発音を再生 →
  やり直し → 送信** の確認ステップを挟む。認識はcontinuousモード＋自動再開で誤終了を防止。
  録音音声はブラウザ内のみで扱い、サーバーには送信しない（都度破棄）
- **総合フィードバック**: 会話終了時にスコア・文法修正・より自然な表現・流暢さ講評（日本語）
- **練習記録**: ストリーク（連続日数）・セッション履歴（レベル付き）
- **学習パス**: カテゴリ別のシナリオ一覧（Skill Treeの土台）

UIはMinimalist-Luxeテーマ（オフホワイト×セージグリーン×ゴールド、明朝体見出し、ラインアートアイコン）。
絵文字は使わない方針。

シナリオのフレーズ・会話はすべてGeminiが動的生成（著作権フリー・追加が容易）。
シナリオを増やすには [app/scenarios.py](app/scenarios.py) の `SCENARIOS` にエントリを足すだけ。

### バックログ（未着手・優先度は状況次第）

- 弱点の復習リスト: 頻出ミスの傾向を「次に復習すべき表現」として提示する
- 通勤用クイック練習モード: 1シナリオのキーフレーズ3〜5個だけをテキストで確認する軽量モード

## セットアップ（ローカル）

```powershell
# 1. 依存のインストール
.venv\Scripts\pip install -r requirements.txt

# 2. APIキーの設定
copy .env.example .env
# .env を開いて GEMINI_API_KEY を設定（https://aistudio.google.com/apikey で無料発行）

# 3. 起動
.venv\Scripts\uvicorn app.main:app --reload
```

ブラウザで http://127.0.0.1:8000 を開く（**Chrome推奨** — 音声認識がWeb Speech API依存のため）。

## Renderへのデプロイ（無料・カード不要）

1. このリポジトリをGitHubにpush
2. [Render](https://render.com) にGitHubアカウントでサインアップ
3. **New → Web Service** でリポジトリを選択（`render.yaml` を自動検出）
4. 環境変数 `GEMINI_API_KEY` をダッシュボードで設定
5. デプロイ完了後、発行されたURLにアクセス（スマホからもOK）

### 無料枠の注意点

- **コールドスタート**: 無操作15分でスリープし、次回アクセスに数十秒かかる
- **ディスクが揮発性**: `TURSO_DATABASE_URL` を設定しない場合、練習履歴・ストリークは
  **再デプロイやスリープ復帰で消える**（下記「データの永続化（Turso）」参照）
- **Gemini無料枠**: Google Cloud側でプロジェクトに課金を有効化すると無料枠が消えるため、
  Geminiのプロジェクトは無課金のまま維持すること

## データの永続化（Turso）

Render無料枠はディスクが揮発性のため、練習履歴・ストリークをPC・スマホ間で共有しつつ
再デプロイ後も残したい場合は、SQLiteの保存先を[Turso](https://turso.tech)（SQLite互換の無料DB、
カード登録不要）に切り替える。設定しなければ今まで通りローカルSQLiteファイルで動作する
（`app/db.py` が環境変数の有無で自動的に切り替える）。

1. https://turso.tech でアカウント作成（GitHubログイン可、カード不要）
2. CLIまたはWebダッシュボードでデータベースを作成
   ```
   turso db create lingoflow
   turso db show lingoflow --url          # → TURSO_DATABASE_URL
   turso db tokens create lingoflow       # → TURSO_AUTH_TOKEN
   ```
3. `.env`（ローカル）またはRenderの環境変数に `TURSO_DATABASE_URL` / `TURSO_AUTH_TOKEN` を設定
4. アプリを再起動すると、以降のデータはTurso側に保存される

**注意**: Turso連携は実際のTursoデータベースを使った動作確認ができていない
（ローカルSQLiteでのみ検証済み）。設定後は一度アプリを操作し、練習記録が正しく保存・表示されるか
確認することを推奨する。Turso無料枠は長期間アクセスがないとDBが一時休止することがあるが、
Render同様「復帰に数秒〜数十秒かかる」程度でデータが消えるわけではない。

`libsql-client`（Turso接続に使うパッケージ）はPython 3.14の`sqlite3`モジュールの変更に
未対応のため、`TURSO_DATABASE_URL`設定時のみ遅延importする実装にしている
（Turso未使用時はこのパッケージを一切読み込まない）。念のためRenderのPythonバージョンも
`.python-version`ファイルで`3.13.4`に固定している。

## 技術構成

| レイヤー | 技術 |
|---|---|
| フロントエンド | バニラJS SPA（レスポンシブ、ビルド不要） |
| 音声認識/合成 | Web Speech API（ブラウザ標準・無料） |
| AI | Gemini API（`gemini-2.5-flash`、REST直叩き） |
| バックエンド | Python / FastAPI |
| DB | SQLite（ローカル）／ Turso（本番、任意） — シナリオキャッシュ＋練習履歴 |
| ホスティング | Render.com 無料Webサービス |

APIキーはバックエンドの環境変数でのみ保持し、フロントエンドには一切渡らない。

## 主なファイル

```
app/
  main.py       # FastAPI ルート定義
  gemini.py     # Gemini API連携（シナリオ生成・会話・フィードバック）
  scenarios.py  # シナリオのシード定義（場面設定のみ。追加はここ）
  expressions.py # 表現集のシード（カテゴリ・頻度付き。追加はここ）
  db.py         # 永続化（SQLite/Turso切替、キャッシュ・履歴・表現集・ストリーク計算）
static/
  index.html / style.css / app.js   # フロントエンド一式
render.yaml     # Renderデプロイ設定
```
